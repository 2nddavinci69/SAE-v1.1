### Contributing to SAE-v1.1
Thank you for your interest in contributing to the Sovereign AI Execution (SAE-v1.1) Open Reference Standard. This project relies on community collaboration to maintain a secure, deterministic, and governance-driven execution standard.

### How to Contribute
1. Reporting Issues
If you encounter bugs in the reference implementation or identify gaps in the documentation, please open an Issue. Please ensure you:

* ** Clearly describe the issue.
* ** Specify the version of the standard you are referencing.

2. Proposing Changes
If you wish to suggest modifications to the standard:

* ** Fork the repository and create a new branch.
* ** Ensure all proposed changes align with the core Trust Model (Identity, Governance, Runtime, and Audit).
* ** Verify that any new implementation logic adheres to the required cryptographic standards (FIPS 203, FIPS 204, and SHA-3).
* ** Submit a Pull Request for review.

3. Governance
All contributions are subject to the project's Governance Model, which prioritizes technical integrity and security over feature additions. Contributions that weaken the security posture of the standard or deviate from the normative requirements will not be accepted.
